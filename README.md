# 🛠 創客社教學專案 GitHub 使用說明（給教學幹部）

**一、我們用 GitHub 做什麼？**

這個 Repo 主要用來：

紀錄你的 教學專案構想

規劃 材料、時程、Demo

讓大家能快速了解目前進度

避免「做到一半才發現做不出來」

👉 一個教學專案 = 一個 Issue


**二、使用說明**
✅ Issue = 你的開發日記
✅ Project Board = 進度看板
✅ Status = 目前做到哪一步



**三、第一次使用：請照以下 5 個步驟**


Step 1️⃣ 建立你的教學專案（Issue）

到上方點 Issues

點 New issue

填寫專案內容（可以用模板，看得出來你做到哪遇到甚麼問題就好）

按 create

👉 建立完成後，你的教學專案紀錄就存在了


Step 2️⃣ 把 Issue 加進 Project

在 Issue 頁面右側：

找到 Projects

選擇"進度規劃"


Step 3️⃣ 設定「你是誰」

Projects 旁有個下拉符號 v 點進去

找到教學幹部：選你的名字

Status：選目前狀態（一開始通常是 Planning / In Review, 如果只有想法但沒有細節請先放在Backlog）

⚠️ 沒選名字 = 不知道這是誰的專案，不被納入開發考量。有興趣認領專案的人可以自行協調，否則會被我規畫為留給下屆開發參考。

⚠️ 選了名字 = 是這個專案的主要負責人，不代表你要自己做，負責權可以自行協調更換。


Step 4️⃣ 更新進度（每週至少一次）

例如以下至少一種即可：
勾選 checklist
新增留言說明進度
上傳照片 / 圖片
更新 Status
其他...


**四、狀態說明（改變方式:project board裡拖卡片或在issue裡改）**

當進度有變化時，請更新 Status：

求救 → 卡關或太忙,人生低潮需要幫忙

Backlog → 專案存在想法階

Planning → 專案初步計畫 (需要的材料，預計的功能，開發期程規劃...)

In progress → 開發中

Demo Ready → 基本功能運作正常 (到checkpoint時一定要做到)

Done → 可用於社課

**Demo Ready, In progress, 求救 在專案能現身社課前會反反覆覆來回，不同開發版本都是inprogress，出版及最新版(覺得能用在社課)時都是demo ready要找教學長和其他教學一起驗可行性**


五、遇到卡關怎麼辦？

請在 Issue 裡填寫 「需要支援 / 卡關」，例如：

材料不確定, 程式跑不起來, 硬體接線有問題

👉 寫出來 = 其他教學才能幫你
👉 不寫 = 大家都以為你很順（出事 = 承擔教學長的地獄怒火）


**六、重要規則（請遵守）**

❗ 不要一個專案開多個 Issue

❗ 不要放著不更新紀錄

❗ Status 請反映真實狀態


**七、最後一句話**

自己喜歡，社員喜歡即可

有問題就寫，卡關就說，
