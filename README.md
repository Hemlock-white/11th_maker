# 🛠 創客社教學專案 GitHub 使用說明（給教學幹部）



一、我們用 GitHub 做什麼？

這個 GitHub 主要用來：

紀錄你的 教學專案構想

規劃 材料、時程、Demo

讓大家能快速了解目前進度

避免「做到一半才發現做不出來」

👉 一個教學專案 = 一個 Issue

二、使用說明
✅ Issue = 你的開發日記
✅ Project Board = 進度看板
✅ Status = 目前做到哪一步
三、第一次使用：請照以下 5 個步驟
Step 1️⃣ 建立你的教學專案（Issue）

到上方點 Issues

點 New issue

填寫專案內容（可以用模板，看得出來你做到哪遇到甚麼問題就好）

按 create

👉 建立完成後，你的教學專案紀錄就存在了

Step 2️⃣ 把 Issue 加進 Project

在 Issue 頁面右側：

找到 Projects

確認有在"進度規劃"

Step 3️⃣ 設定「你是誰」



教學幹部：選你的名字

Status：選目前狀態（一開始通常是 Planning）

⚠️ 沒選名字 = 教學長不知道這是誰的專案

Step 4️⃣ 更新進度（每週至少一次）

只要做其中一種即可：

勾選 checklist

新增留言說明進度

上傳照片 / 圖片

更新 Status

👉 有更新 = 有在做
👉 沒更新 = 視為停滯

Step 5️⃣ 改變狀態（拖卡片）

當進度有變化時，請更新 Status：

Planning → Building（開始做硬體）

Coding → Integration（軟硬體開始合）

Demo Ready → Done（教學已完成）

四、Status 狀態說明（一定要看）
狀態	說明
Backlog	想法階段，尚未規劃
Planning	教案、材料、流程規劃
Building	硬體製作、接線
Coding	程式撰寫
Integration	軟硬體整合（重要）
Demo Ready	已可完整展示
Done	教學完成
五、遇到卡關怎麼辦？

請在 Issue 裡填寫 「需要支援 / 卡關」，例如：

材料不確定

程式跑不起來

硬體接線有問題

👉 寫出來 = 教學長才能幫你
👉 不寫 = 大家都以為你很順（但其實不是）

六、重要規則（請遵守）

❗ 不要一個專案開多個 Issue

❗ 不要放著不更新

❗ Status 請反映真實狀態

這不是在考核你，是在避免大家一起翻車。

七、你不需要做的事

❌ 不用學 Git 指令

❌ 不用寫複雜程式

❌ 不用管別人的 Issue

👉 顧好你自己的那一張卡就好

八、最後一句話

教學專案不是比誰厲害，
是比誰「能被教、能被救、能 Demo」。

有問題就寫，卡關就說，
我們一起把教學做完 💪
